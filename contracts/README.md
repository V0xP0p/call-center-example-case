# Shared Contracts

These schemas are intentionally neutral and example-oriented. They define the minimum shared shapes required so separate asset teams can produce compatible examples and tests against the same use-case pack.

The schemas in this directory are not a mandate for internal implementation details. They are the interoperability layer for fixtures, traces, and acceptance cases.

## Included Schemas

- `route-decision.schema.json`
- `tool-contracts.schema.json`
- `retrieval-contracts.schema.json`
- `guardrail-decision.schema.json`
- `eval-case.schema.json`
- `run-trace.schema.json`
